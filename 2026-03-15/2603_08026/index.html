Summary of DyLLM: Efficient Diffusion LLM Inference via Saliency-based Token Selection and Partial Attention

1. Overview

Masked Diffusion Language Models (MDLMs) enable parallel token decoding, a promising alternative to slow sequential autoregressive LLM generation, but suffer from high computational overhead: their iterative denoising process reprocesses the entire sequence at every step, unlike autoregressive models that leverage incremental KV caching. The key insight motivating this work is that most token representations remain stable across consecutive diffusion steps, with only a small subset of "salient tokens" undergoing meaningful changes that contribute to generation quality. DyLLM is a training-free inference framework that exploits this temporal sparsity to accelerate MDLM decoding by selectively computing only salient tokens, while reusing cached activations for stable non-salient tokens. It delivers up to 9.6x higher throughput on state-of-the-art MDLMs (LLaDA, Dream) while preserving or improving baseline accuracy across reasoning, code, and general knowledge benchmarks.

2. Key Contributions

  • Layer-adaptive saliency mechanism: A dynamic, threshold-based policy that identifies salient tokens per layer using cosine similarity of attention contexts between adjacent denoising steps, eliminating redundant Feed-Forward Network (FFN) computation for stable tokens without retraining.
  • Saliency-aware approximate attention: A low-complexity attention implementation that reduces quadratic sequence length overhead by computing exact attention only for salient tokens, while approximating context updates for non-salient tokens using cached values with provable error bounds.
  • Scalable training-free framework: Evaluated across LLaDA 8B and Dream 7B MDLMs, DyLLM achieves up to 7.6x and 9.6x throughput improvements respectively across diverse benchmarks, outperforming state-of-the-art MDLM acceleration baselines (Fast-dLLM, dLLM-Cache) while maintaining or improving generation quality.

3. Method

DyLLM operates in two phases: an initial warmup phase of 4 full sequence passes to initialize caches, followed by an accelerated refinement phase that only processes salient tokens per layer and step.

Across consecutive diffusion steps, the majority of token representations remain stable; only a small subset, which we term salient tokens, undergoes meaningful transitions that contribute to the next update.

3.1 Motivation and Temporal Sparsity Observation

The per-step compute overhead of baseline MDLMs is illustrated in Figure 1, which shows that MDLM runtime is dominated by repeated full-sequence FFN and attention computation, unlike autoregressive LLMs that have low per-step overhead after prefill.

To quantify representation stability, the authors define temporal cosine similarity for token $i$ at layer $l$, step $t$:

$$

s_{t,l}^{(i)} = \frac{C_{t,l}^{(i)} \cdot C_{t-1,l}^{(i)}}{\|C_{t,l}^{(i)}\| \|C_{t-1,l}^{(i)}\|}

$$

where $C_{t,l}^{(i)}$ is the attention context vector for the token. This metric measures directional alignment of attention contexts between consecutive steps, with values near 1 indicating nearly identical representations.

As shown in Figure 2, most tokens have $s_{t,l}^{(i)} \approx 1$ across all layers, with stability higher in early layers and more variation in deeper layers. 3.2 Saliency Detection and FFN Optimization

Salient tokens are defined as $\mathcal{A}_{t,l} = \{i \mid s_{t,l}^{(i)} \leq \tau\}$, where $\tau$ is a configurable threshold. Non-salient tokens skip FFN computation entirely, reusing cached activations from the previous step. Two formal propositions validate this approach:

  1. Scale Invariance under Linear Projection:

$$

\text{RMSNorm}((\alpha C) W_o) = \text{RMSNorm}(C W_o)

$$

This shows that only the direction (not magnitude) of the attention context matters for downstream FFN input, validating cosine similarity as a suitable saliency metric.

  1. Error Bound via Directional Alignment:

$$

\delta \leq \kappa(W_o)\sqrt{2(1 - s_{t,l})}

$$

where $\delta$ is the approximation error from reusing cached FFN output, and $\kappa(W_o)$ is the condition number of the attention output projection matrix. This confirms that error is bounded by the cosine similarity, so reusing outputs for high-similarity tokens introduces negligible error.

3.3 Saliency-Aware Approximate Attention

To reduce the quadratic complexity of full attention, the authors decompose the temporal change in attention context:

$$

\Delta C_{t,l} = (S_{t-1, l} + \Delta S)\Delta V + (\Delta S)V_{t-1,l}

$$

where $\Delta S$ and $\Delta V$ are changes in attention scores and value matrices between steps. The dual-path approximate attention mechanism uses this decomposition (Figure 5):

  • Salient Path: For tokens $i \in \mathcal{A}_{t,l}$, compute exact full attention to capture all context updates.
  • Non-salient Path: For tokens $i \notin \mathcal{A}_{t,l}$, approximate updates as $S_{t,l}^{(i,\cdot)} \Delta V_{t,l}$, only incorporating changes from salient tokens, reducing complexity from $O(N^2d)$ to $O(N|\mathcal{A}_{t,l}|d)$.

3.4 Response-Only Optimization

Leveraging the locality bias of Rotary Positional Embeddings (RoPE), DyLLM prioritizes response tokens for saliency checks, as stable prompt tokens rarely become salient. Full sequence processing is only done periodically (every 4 steps) to refresh prompt context, with only salient tokens processed even during these refresh steps.

4. Experiments

All experiments were run on a single NVIDIA H100 80GB GPU, evaluating LLaDA 8B and Dream 7B instruct models against baselines: original MDLM implementation, Fast-dLLM, and dLLM-Cache. Benchmarks include mathematical reasoning (GSM8K, MATH), general knowledge (MMLU-pro), and code generation (MBPP).

4.1 Main Results

DyLLM consistently outperforms all baselines in throughput while preserving or improving accuracy:

  • For LLaDA 8B: 4.57x to 7.6x throughput gain, with accuracy matching or exceeding baseline across all benchmarks.
  • For Dream 7B: 6.6x to 9.6x throughput gain, with up to 4% accuracy improvement on GSM8K and MATH benchmarks.

4.2 Ablation Studies

Ablations validate the impact of the similarity threshold $\tau$ on accuracy and throughput:

  • Lower $\tau$ selects fewer salient tokens, increasing throughput but potentially introducing minor accuracy loss, while higher $\tau$ is more conservative.
  • The optimal $\tau$ (0.99 for LLaDA, 0.995 for Dream) delivers near-maximal throughput with negligible or positive accuracy impact, as shown in the ablation plots for GSM8K/MBPP (Figure 8) and MATH/MMLU-pro (Figure 9):

DyLLM also scales better with higher parallel decoding degrees ($n_u$, number of tokens unmasked per step) than baselines, as it avoids the rigid refresh overhead of fixed-schedule caching methods.

5. Limitations & Future Work

  • Limitations: DyLLM requires custom CUDA kernels for maximal performance, and the optimal similarity threshold $\tau$ may need minor tuning for new model architectures or datasets. It is currently evaluated only on offline batched inference settings.
  • Future Work: The authors plan to extend DyLLM to larger MDLMs, integrate it with confidence-aware parallel decoding schemes to further boost throughput, optimize it for low-latency online inference, and combine it with speculative decoding techniques for hybrid autoregressive-diffusion generation pipelines.
