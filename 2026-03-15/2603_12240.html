1. Overview

Problem Addressed

Existing diffusion model acceleration techniques (token merging, downsampling) are optimized exclusively for generation quality, and disproportionately degrade discriminative performance when the same diffusion backbone is reused for classification tasks. This creates a gap between compression schemes that produce visually acceptable outputs and those that preserve the fine-grained cues needed for recognition, limiting deployment of dual-purpose generative systems.

Key Insight

The core insight is frequency separation: intermediate feature signals can be disentangled into high-frequency components (edges, textures, critical for classification) and low/mid-frequency components (global shapes, semantic layouts, critical for generation). A balanced spectral retention policy that preserves both frequency bands enables compression that supports both tasks without sacrificing performance on either.

2. Key Contributions

  • BiGain reframes diffusion token compression as a bi-objective optimization problem (preserving both generation fidelity and discriminative utility) and delivers a training-free, plug-and-play implementation compatible with all common diffusion backbones.
  • First work to jointly advance both generation and classification performance under diffusion acceleration.
  • Provides generalizable frequency-aware design guidance for token compression: merge only spectrally smooth low-frequency tokens, downsample keys/values while retaining full-resolution queries, to support low-cost deployment of dual-purpose generative models.

3. Method

BiGain implements the balanced spectral retention principle via two complementary, training-free operators:

3.1 Laplacian-Gated Token Merging (L-GTM)

The operator prioritizes merging of low-frequency smooth tokens while avoiding merges of high-contrast, detail-rich tokens. The exact stated goal is:

"In this part, we aim to merge spectrally smooth tokens while discouraging merges of high-contrast tokens."

First, hidden tokens $\mX \in \mathbb{R}^{N\times d}$ are reshaped to spatial form $\mX \in \mathbb{R}^{H\times W\times C}$, and a per-location frequency score is computed:

$$\mF=\operatorname{Reduce}_c\!\bigl(|\mX * \mL|\bigr),\quad

\mL=\begin{bmatrix}0&1&0\\[2pt]1&-4&1\\[2pt]0&1&0\end{bmatrix},\quad

\mF\in\mathbb{R}^{H\times W}$$

Where:

  • $\mL$ is the 2D Laplacian kernel, a second-order derivative operator measuring local contrast between a pixel and its neighbors
  • $\operatorname{Reduce}_c$ is channel-wise aggregation (mean or $\ell_2$ norm) across feature channels
  • Higher values of $\mF$ indicate higher-frequency, more discriminative tokens that should not be merged.

Merging proceeds by selecting tokens with the lowest $\mF$ scores as merge destinations, merging the top $r\%$ most similar source-destination token pairs via equal averaging, and retaining high-$\mF$ tokens intact. This reduces attention FLOPs from $\mathcal{O}(N^2d)$ to $\mathcal{O}(N'^2d)$ where $N' < N$ is the reduced token count.

3.2 Interpolate-Extrapolate KV Downsampling (IE-KVD)

This operator reduces attention compute by downsampling only keys (K) and values (V) while retaining full-resolution queries (Q) to preserve attention precision. The stated goal is:

"In this component, we aim to reduce attention cost by downsampling keys/values while keeping queries intact to preserve localization and alignment."

The downsampling operator is a controllable mix of nearest-neighbor pooling (high-frequency preserving) and average pooling (low-frequency preserving):

$$\mathcal{D}_{\alpha,s}(\mZ)[i]

= \alpha\,\mZ[\text{nearest}(i)]

  • (1{-}\alpha)\,\tfrac{1}{|\mathcal{N}_s(i)|}

\sum_{j \in \mathcal{N}_s(i)} \mZ[j]$$

Where:

  • $\alpha$ is the interpolation weight, tunable between values <1 (extrapolate to prioritize average pooling) and >1 (extrapolate to prioritize nearest pooling)
  • $s$ is the downsampling stride, $\mathcal{N}_s(i)$ is the local spatial neighborhood for position $i$
  • The operator is applied only to K and V, while Q remains full-resolution.

This reduces attention FLOPs to $\mathcal{O}(N\tilde{N}d)$ where $\tilde{N}$ is the downsampled K/V length, and preserves fine-grained receptive fields for all output tokens.

Both operators are timestep-local and do not use cross-timestep caching, making them fully compatible with diffusion classifier Monte Carlo sampling protocols.

4. Experiments

Setup

Tests are run on two standard diffusion backbones: Stable Diffusion 2.0 (U-Net based) and DiT-XL/2 (Transformer based), using public pretrained weights. Datasets include ImageNet-1K, ImageNet-100, Oxford-IIIT Pets, and COCO 2017. Metrics: Top-1 accuracy/macro mAP for classification, FID for generation, and FLOPs for efficiency measurement.

Key Results

  • On Oxford-IIIT Pets at ~10% FLOP reduction, BiGain_TM (merging) reduces classification accuracy by only 2.65% vs. baselines (ToMe, DiP-GO, SiTo, MosaicDiff) that drop 3.65-12.19% accuracy. BiGain_TD (downsampling) reduces accuracy by only 1.13% vs. ToDo's 1.88% drop.
  • On ImageNet-1K with 70% token merging on Stable Diffusion 2.0, BiGain improves classification accuracy by 7.15% over ToMe, while also improving generation FID by 0.34 (1.85%).
  • As downsampling factor increases from 2x to 8x, BiGain's classification advantage over ToDo grows from 0.22% to 9.87% on Oxford-IIIT Pets, with FID improved by up to 2.27.
  • On DiT-XL/2, the ToDo baseline collapses completely at 3x downsampling (FID jumps to 190.18) while BiGain remains stable, outperforming average pooling on both classification and generation.

6. Limitations & Future Work

  • Current implementation uses fixed compression schedules across timesteps; adaptive scheduling that adjusts merge/downsample ratios based on timestep noise levels could further improve performance.
  • The method is only tested on image diffusion models; future work will extend it to video, text, and multi-modal diffusion systems.
  • The interpolation weight $\alpha$ for IE-KVD is manually tuned; data-driven adaptive $\alpha$ selection per layer/timestep could yield better task-specific trade-offs.
  • Performance degrades at compression ratios above 70% merge / 8x downsampling; future work will explore improved spectral retention mechanisms for extreme sparsity regimes.
