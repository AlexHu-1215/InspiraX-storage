Summary of BiGain: Unified Token Compression for Joint Generation and Classification

1. Overview

Problem Addressed

Existing diffusion model acceleration methods (token merging, downsampling) exclusively optimize for generation fidelity under reduced compute, but severely degrade the latent discriminative capacity of diffusion backbones used for classification tasks. This creates a critical gap for dual-purpose generative systems that support both generation and classification (e.g., medical imaging, industrial inspection, safety-critical perception) where a single backbone is used for both tasks.

Key Insight

The core insight is frequency separation: mapping intermediate feature-space signals to a frequency-aware representation disentangles high-frequency details (edges, textures critical for classification) from low/mid-frequency semantics (shapes, layouts critical for coherent generation). Compression that follows a balanced spectral retention rule (preserving both frequency bands) can maintain both generation quality and classification accuracy without tradeoff. The resulting framework, BiGain, is a training-free, plug-and-play solution that works for both U-Net and DiT diffusion backbones.

2. Key Contributions

  • Frames diffusion token compression as a bi-objective optimization problem (prioritizing both generation fidelity and classification accuracy) instead of the standard single-objective generation-only optimization, with a practical training-free, drop-in solution.
  • First framework to jointly advance both generation and classification performance under diffusion acceleration, enabling low-cost deployment of dual-purpose generative systems.
  • Introduces two frequency-aware compression operators: Laplacian-gated token merging (L-GTM) and Interpolate-Extrapolate KV Downsampling (IE-KVD), and provides empirically validated design guidance for future token compression work: merge only spectrally smooth regions, and preserve full-resolution queries while downsampling keys/values.

3. Method

BiGain follows the core design rule:

"balanced spectral retention: preserve high-frequency detail (edges/textures) and low/mid-frequency content (global semantics)"

3.1 Laplacian-Gated Token Merging (L-GTM)

This operator merges only low-contrast, spectrally smooth tokens while preserving high-contrast, detail-rich tokens critical for classification.

  1. Frequency Score Calculation: A 2D Laplacian kernel is applied to hidden state tokens to measure local contrast (proxy for frequency content):

$$\mF=\operatorname{Reduce}_c\!\bigl(|\mX * \mL|\bigr),\quad

\mL=\begin{bmatrix}0&1&0\\[2pt]1&-4&1\\[2pt]0&1&0\end{bmatrix},\quad

\mF\in\R^{H\times W}$$

  • $\mX$: Hidden state tensor reshaped to spatial dimensions $H \times W$ with $C$ channels
  • $\mL$: Standard Laplacian kernel approximating second-order spatial derivatives, measuring local contrast
  • $\operatorname{Reduce}_c$: Channel-wise aggregation (mean or $\ell_1$ norm)
  • $\mF$: Frequency score map, where higher values indicate higher local contrast/higher frequency content
  1. Gated Merging: Tokens with the lowest $\mF$ scores (smoothest regions) are selected as destination anchors, while remaining tokens form the source set. The top $r\%$ most similar source-destination pairs are merged via equal averaging. Class, time, and text conditioning tokens are never merged to avoid performance degradation. A faster Adaptive Block Merging variant merges entire $s \times s$ blocks if their maximum frequency score is below a threshold, for additional efficiency.

3.2 Interpolate-Extrapolate KV Downsampling (IE-KVD)

This operator reduces attention compute by downsampling keys (K) and values (V) while preserving full-resolution queries to retain attention precision:

"keeping queries at full resolution preserves the model’s ability to localize and attend precisely, whereas the KV shrinkage reduces memory and FLOPs smoothly"

The downsampling operator is defined as:

$$\mathcal{D}_{\alpha,s}(\mZ)[i]

= \alpha\,\mZ[\text{nearest}(i)]

  • (1{-}\alpha)\,\tfrac{1}{|\mathcal{N}_s(i)|}

\sum_{j \in \mathcal{N}_s(i)} \mZ[j]$$

  • $s$: Downsampling stride
  • $\alpha$: Controllable weight interpolating between nearest-neighbor pooling ($\alpha=1$, prioritizes high-frequency detail) and average pooling ($\alpha=0$, prioritizes low-frequency smoothing)
  • $\mathcal{N}_s(i)$: Set of spatial neighbors of position $i$ in the original grid

$\tilde{\mK} = \mathcal{D}_{\alpha,s}(\mK)$ and $\tilde{\mV} = \mathcal{D}_{\alpha,s}(\mV)$ while queries remain full size, reducing attention cost from $\mathcal{O}(N^2d)$ to $\mathcal{O}(N\tilde{N}d)$ ($\tilde{N}$ = number of downsampled KV tokens) with no unmerging step needed, as output shape remains $N$.

Both operators are timestep-local with no cross-timestep caching, making them fully compatible with diffusion classification's paired Monte Carlo sampling rule without biasing scores.

4. Experiments

Setup

  • Backbones: Stable Diffusion 2.0 (U-Net-based) and DiT-XL/2 (Transformer-based) with official pre-trained weights
  • Datasets: ImageNet-1K, ImageNet-100, Oxford-IIIT Pets, COCO 2017
  • Metrics: Classification (Top-1 accuracy, mAP for multi-label COCO); Generation (FID); Efficiency (FLOPs reduction, merge/downsampling ratio)

Key Results

  • On Oxford-IIIT Pets with ~10% FLOPs reduction: BiGain_TM (merging) has only 2.65% classification accuracy drop vs. no acceleration, outperforming ToMe (8.07% drop), DiP-GO (4.5% drop), SiTo (12.19% drop) and MosaicDiff (3.65% drop). BiGain_TD (downsampling) has a 1.13% drop vs. ToDo's 1.88% drop.
  • On ImageNet-1K with 70% token merging on Stable Diffusion 2.0: BiGain improves classification accuracy by 7.15% over ToMe, while also improving generation FID by 0.34 (1.85%).
  • On DiT-XL/2 with 3x downsampling: ToDo fails catastrophically (classification accuracy drops to 8.46%) while BiGain retains stable performance (61.58% accuracy) and far better generation FID.
  • Performance gains grow with more aggressive compression: BiGain delivers up to 9.87% classification accuracy gain and 2.27 FID improvement on Pets at 8x downsampling vs. ToDo.
  • Ablations confirm: (1) Compressing only self-attention layers delivers the best tradeoff, vs. compressing cross-attention or MLP layers; (2) Laplacian $\ell_1$ frequency scoring outperforms all other token scoring heuristics (norms, variance, DFT metrics, cosine similarity) by 0.3-1.9% across merge ratios.

6. Limitations & Future Work

Limitations

  • BiGain is currently validated only on 2D image diffusion models, with no testing on video, 3D, or text diffusion backbones.
  • The current implementation uses fixed compression schedules across all timesteps, rather than adapting to the varying noise level at different denoising steps.

Future Work

  • Extend the frequency-aware compression principle to video, 3D, and multi-modal diffusion models.
  • Integrate BiGain with step-reduction acceleration methods (DDIM, DPM-Solver) to achieve further end-to-end speedups.
  • Design dynamic compression schedules that adapt merge/downsampling ratios per timestep based on noise level to optimize the speed-performance tradeoff.
  • Optimize BiGain operators for edge hardware deployment to enable low-latency dual-purpose generative systems on resource-constrained devices.
